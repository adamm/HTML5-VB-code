Font-face Selector

Synopsis

<style>
  @font-face {
    font-family: MyFont;
    src: url(myfont.ttf), url(myfont.eot);
  }
  selector {
    font-family: MyFont;
  }
</style>

Description

No longer are you limited to so-called web safe fonts, the short list of fonts built-in to browsers and defined in HTML 4.01. HTML5 web browsers now support a new feature in CSS3, custom fonts. Now you can actually import your own TrueType (TTF), OpenType (OTF), and Embedded OpenType (EOT) font files into your web page, and display text with them.

Example

The @font-face users a selector-like syntax that you can use and re-use, once for each new font you introduce. The font-family property assigns a proper name to the font, and is used later when assigning the font with a CSS selector.

The files referenced by the src property must be stored on your web server, just like image and HTML files. They will be downloaded by the web browser and installed onto the userâ€™s computer, but only within the context of the web browser session.

Multiple fonts formats are defined by src because different browsers support different formats. Since Internet Explorer only supports EOT, and Chrome only TTF, defining both ensures maximum support.
