Scrollbar Pseudo-selector

Synopsis

<style>
  selector {
    overflow: auto;
    display: block;
    width: windowwidth;
    height: windowheight;
  }
  ::webkit-scrollbar {
    width: scrollbarwidth;
    height: scrollbarheight;
  }
  ::webkit-scrollbar-button {
  }
  ::webkit-scrollbar-thumb {
  }
  ::webkit-scrollbar-track {
  }
  ::webkit-scrollbar-track-piece {
  }
</style>

Description

You can create a custom scrollbar on any display block that has the overflow property set. This can be used to replace the bland scrollbar that is built-in to the browser’s operating system. This feature is supported in only in desktop WebKit browsers, like Chrome and Safari browsers, but is not available in FireFox 4, Internet Explorer 9, or Opera 11. This is because WebKit introduced and implemented the feature, and it is not yet included by the W3C for CSS3.

Example

The windowwidth and windowheight are length values indicating the size of the scrollable window you want to create. The scrollbarwidth and scrollbarheight represent the maximum width of the vertical scrollbar, and maximum height of the horizontal scrollbar. Now you can start defining the style of the main scrollbar components: the buttons, the movable bar, and the bar’s track.

These scrollbar style selectors accept additional pseudo-classes which define the style of scrollbar sub-components. For example, you can extend ::-webkit-scrollbar-button by adding :vertical or :horizontal into the selector to stylize the scrollbar axis differently; then create another rule and add :increment or :decrement to stylize the direction buttons differently. This will allow you to produce unique up, down, left, and right arrow buttons.

Creating the scrollbar first requires you to define the basic CSS structure, than apply the images now. Therefore, it is easier to begin with basic CSS border and background-color properties to help you understand how the selectors function, and add images later.
