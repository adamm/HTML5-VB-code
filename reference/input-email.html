Input Email Type

Synopsis

<input type='email' name='field' value='email' multiple>

Description

You can use the email input type to restrict input fields to values formatted like email addresses only. Supported web browsers will reject values that are not formatted with an “@”-sign and a specific domain name with a TLD, or top-level domain, like user@domain.com. Or to be precise, values that do not follow RFC5322.

Note this feature makes no effort to verify that the user owns the email address provided, or whether it is even real. If you require this functionality, you will have to create a server-side CGI script that receives this field value and actually emails the user a secret verification code or URL for subsequent follow-up.
